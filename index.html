Revertir un commit no deseado

Opcion 1

git commit --amend usarlo siempre y cuando aun estas en preparacion

Opcion 2

git reset --hard 'nameCommit or id' elimina todos los archivos generados

Git checkout

los is de los commit funcionan como ramas para desplazarse si es que no se ha creado una rama inicial

Git push --force

Si se ha realizado conflictos sin retorno a pequeña escala es opcional. no aconsejable para proyectos grandes. Busca otra opcion de solucionar los commit no deseados

Git branch

Uno de los puntos a favor es que los cambios de la rama no afectan al trabajo que estés anteriormente Nota: si se ha hecho una modificacion antes de crear la rama, solo se mostrara en la rama creada mas no en la que se inicio

(git checkout -b) crea y posiciona en la rama nombrada

(git merge "nameBranch") integra la rama creada con la principal la publicacion de una rama se puede realizar desde la misma rama y aceptar el merge directamente desde la pagina de github
